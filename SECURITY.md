# Security Policy

Security and compliance are the core of Lemma. We take the security of this project and its users extremely seriously.

## Supported Versions

Currently, the project is in a pre-release state (Phase 0/1 development). Security updates will be comprehensively supported starting from the `v1.0.0` release.

| Version | Supported          |
| :------ | :----------------- |
| `1.0.x` | To be determined   |
| `< 1.0` | ❌ Not Supported   |

## Submitting a Vulnerability

**DO NOT OPEN A PUBLIC ISSUE FOR SECURITY REPORTS.**

Reporting a vulnerability publicly compromises the security of the project and its users before a patch can be developed. 

Please report any security vulnerabilities via **[GitHub Security Advisories](https://github.com/JoshDoesIT/Lemma/security/advisories/new)** to keep the report private.

### Reporting Guidelines
When submitting an advisory, please include the following information:
- A detailed description of the vulnerability.
- Steps to reproduce the issue.
- The versions of Lemma affected.
- The potential impact or risk (e.g., Remote Code Execution, Privilege Escalation).
- Any proposed mitigations or fixes, if you have them.

## Response Timeline

We commit to the following response timeline for security issues:
- **Acknowledgement:** We will acknowledge your report within 48 hours.
- **Triage:** We will confirm the vulnerability and provide an initial severity assessment within 5 business days.
- **Remediation:** For critical vulnerabilities, we aim to provide a patch or mitigation within 14 days of confirmation. 

We will keep you informed of our progress throughout the remediation process. Once a patch is available and deployed, we will coordinate a public disclosure (CVE assignment if appropriate) and credit you for the discovery.
